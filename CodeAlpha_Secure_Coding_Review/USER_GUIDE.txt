SECURE CODE SCANNER - USER GUIDE
================================================================================

🎯 WHAT IS THIS?
A user-friendly desktop application that scans Python projects for security 
vulnerabilities and provides instant feedback with fixes.

✨ FEATURES:
- Beautiful graphical interface (no command line needed!)
- One-click folder scanning
- Color-coded severity levels (🔴 Critical, 🟠 High, 🟡 Medium)
- Instant results display
- Save reports with timestamp
- Detects 5 common vulnerabilities

🚀 HOW TO RUN:

Method 1 (Easiest):
   - Double-click "Run_Scanner.bat"
   - The application will open automatically

Method 2 (Manual):
   - Open Command Prompt
   - Navigate to folder: cd "c:\Users\srini\OneDrive\Desktop\secuirty coding review"
   - Run: python SecureCodeScanner.py

📖 HOW TO USE:

Step 1: Launch the Application
   - Double-click Run_Scanner.bat or run the Python file

Step 2: Select Folder
   - Click "📁 Browse Folder" button
   - Choose the Python project folder you want to scan
   - Selected path will be displayed

Step 3: Scan
   - Click "🔍 SCAN NOW" button
   - Wait for scanning to complete (usually takes seconds)
   - Results appear automatically

Step 4: Review Results
   - Scroll through the results panel
   - Each vulnerability shows:
     * Severity level with icon
     * File location and line number
     * Vulnerable code snippet
     * Recommended fix

Step 5: Save Report (Optional)
   - Click "💾 Save Report" button
   - Choose location and filename
   - Report saved as text file with timestamp

Step 6: Clear (Optional)
   - Click "🗑️ Clear" button to reset and scan another folder

🔍 WHAT IT DETECTS:

1. 🔴 CRITICAL - eval() Usage
   - Dangerous code execution
   - Fix: Use ast.literal_eval() or json.loads()

2. 🔴 CRITICAL - SQL Injection
   - Database security risk
   - Fix: Use parameterized queries

3. 🟠 HIGH - Hardcoded Passwords
   - Exposed credentials
   - Fix: Use environment variables

4. 🟡 MEDIUM - Weak Hashing (MD5/SHA1)
   - Insecure encryption
   - Fix: Use SHA256 or SHA512

5. 🟡 MEDIUM - Missing Input Validation
   - Potential data issues
   - Fix: Add type checking and sanitization

📊 SAMPLE WORKFLOW:

1. Launch app → Beautiful blue interface opens
2. Browse → Select "test_project" folder
3. Scan → Click scan button
4. Results → See 7 vulnerabilities found
5. Review → Read each issue and fix suggestion

- Python 3.x installed
- tkinter (included with Python)
- No additional packages needed!

🎨 INTERFACE GUIDE:

Header: "🔒 SECURE CODE SCANNER" - Application title
Browse Button: Select folder to scan
Scan Button: Start security analysis
Results Panel: Scrollable area showing all findings
Save Button: Export report to file
Clear Button: Reset for new scan

🆘 TROUBLESHOOTING:

Problem: Application won't start
Solution: Ensure Python is installed (python --version)

Problem: "No folder selected" warning
Solution: Click Browse button first before scanning

Problem: No results shown
Solution: Make sure folder contains .py files

Problem: Can't save report
Solution: Choose a valid location with write permissions

📈 UNDERSTANDING RESULTS:

Format for each finding:
[#] 🔴 SEVERITY - Description
📄 File: path/to/file.py
📍 Line: 15
💻 Code: vulnerable_code_here
✅ Fix: recommended_solution

Example:
[1] 🔴 CRITICAL - Use of eval() function
📄 File: test_project\app.py
📍 Line: 23
💻 Code: result = eval(user_input)
✅ Fix: Use ast.literal_eval() or json.loads() instead

🎓 BEST PRACTICES:

1. Scan before committing code
2. Address all CRITICAL issues immediately
3. Keep reports for audit trails
4. Educate team on secure coding
5. Integrate into development workflow
